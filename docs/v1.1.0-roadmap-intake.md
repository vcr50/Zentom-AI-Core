# v1.1.0 Roadmap Intake

## 1. Purpose
This document captures, prioritizes, and scopes the feature requests and enhancements slated for the first major update (v1.1.0) following the initial General Availability (GA) launch of SentinelFlow.

## 2. Current GA Status
**Status:** LIVE IN PRODUCTION (GA) - Stabilization Phase

## 3. Stabilization Gate
**CTO Rule:** No major v1.1.0 feature development will commence until the platform achieves 14 consecutive days of production stability (Zero new P1/P2 defects). Stabilization takes absolute priority over roadmap velocity.

## 4. Customer Feedback Sources
Candidate items for v1.1.0 have been sourced from:
- Deferred P3/P4 items in the Pilot Issue Register.
- Qualitative findings from the Customer Usage Review (Milestone 42C).
- Requests submitted to `#sentinelflow-prod-support`.

## 5. Candidate Roadmap Items
- Slack bot / conversational incident assistant
- Near-realtime telemetry widgets
- Dashboard filter UX improvements
- Replay export/share capability
- Approval escalation paths
- AI confidence trend metrics
- RBAC / governance dashboard
- Compliance roadmap documentation

## 6. Priority Scoring Model
*Each candidate item is scored on a scale of 1-5 (5 being highest/most intense) across the following dimensions:*
- **Customer Demand:** 1–5
- **Business Value:** 1–5
- **Security Risk:** 1–5 (Higher score = higher risk)
- **Engineering Effort:** 1–5
- **GA Stability Impact:** 1–5 (Higher score = greater threat to core stability)

*Rule of Thumb for v1.1.0: Select high-value, low-risk improvements.*

## 7. In-Scope for v1.1.0
Based on the CTO's direct recommendation, the following items balance high user impact with low stability risk:
1. **Dashboard filter UX improvements:** Low effort, high daily UX value.
2. **Near-realtime telemetry widgets:** Moderate effort, improves operations visibility.
3. **Replay export/share capability:** Low effort, improves cross-team communication.
4. **Approval escalation paths:** Moderate effort, reduces MTTR bottlenecks.

## 8. Out-of-Scope for v1.1.0 (Deferred)
The following highly-requested but complex features are deferred to v1.2.0 or beyond to protect current stability:
- Slack bot integration
- Full autonomous auto-heal
- Digital Twin simulations
- Custom customer AI agents
- Multi-org enterprise rollout

## 9. Security / Compliance Considerations
None of the in-scope v1.1.0 features alter the core authentication model, change PII masking rules, or remove the human-in-the-loop governance gate.

## 10. Engineering Effort Estimate
- Total estimated effort for v1.1.0 scope: **2-3 Sprints**.

## 11. Product Decision Log
- **Decision:** Defer Slack Bot. **Reason:** Requires establishing a new authentication pathway and API surface area, risking current platform stability.

## 12. Final v1.1.0 Recommendation
The engineering team is authorized to begin architectural design for the four In-Scope items (Section 7) *only* after the Stabilization Gate (Section 3) is officially passed.
