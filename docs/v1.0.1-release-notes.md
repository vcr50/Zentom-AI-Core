
- Resolve approved P0/P1/P2 pilot feedback fixes.

## 8. Rollback Notes

Rollback notes:

- Roll back individual fix commits if validation fails and cannot be safely corrected.
- Revert hosted API changes to previous known-good commit or deploy artifact where applicable.
- Revert Salesforce metadata to previous package state where applicable.
- Re-run affected validation after rollback.
- Notify pilot/customer owner if rollback affects pilot timing.

Rollback trigger examples:

- New P0/P1 introduced.
- Security/privacy regression introduced.
- Package deploy/install fails.
- Hosted API or callout path fails.
- Approval/execution or Case creation fails.
- Replay/audit evidence becomes unreliable.
