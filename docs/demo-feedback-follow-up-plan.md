## 4. Issue Severity Definitions
- **P0 — Immediate blocker:** Must fix before the customer can proceed to pilot.
- **P1 — Core workflow issue:** Must fix for general usability, high priority.
- **P2 — Important demo/customer trust issue:** Address to improve confidence in the product.
- **P3 — Nice-to-have improvement:** UX polish, minor workflow enhancements.
- **P4 — Future roadmap idea:** Valid feedback, but not slated for the immediate beta/pilot phases.

## 5. Fix / Follow-up Owner
- **Engineering / Product:** Triage and schedule P0/P1/P2 fixes.
- **Sales / AE:** Manage customer communication and follow-up timelines.
- **Design:** Address P2/P3 UI/UX improvements.

## 6. Target Milestone
All actionable feedback (P0-P2) should be triaged and mapped to an upcoming milestone (e.g., Milestone 36 — Demo Feedback Patch).

## 7. Follow-up Communication Plan
- **Timeline:** Within 24-48 hours post-demo.
- **Channel:** Email thread with primary customer stakeholders.
- **Content:** Thank you, assets (see below), and confirmation of next steps based on the Demo Outcome Decision.
